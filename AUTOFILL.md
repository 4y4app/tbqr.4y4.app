# 入力補完スクリプト仕様

`src/pages/index.astro` の `initAutoFillFromEmail()` が担う、メールアドレス入力に連動した自動補完の挙動をまとめる。

## 目的

メールアドレスと同じ値になることが多い「差出人名（表示名）」「受信ユーザー名」を、メールアドレス入力のたびに毎回手入力する手間を省く。

## 対象フィールド

| フィールド | id | 設置場所 |
|---|---|---|
| トリガー | `#email` | アカウント基本情報 |
| 補完先1 | `#displayname` | アカウント基本情報 |
| 補完先2 | `#username` | 受信サーバー (Incoming) |

## 動作ルール

1. `#email` に `input` イベントが発生するたびに、現在の入力値を `#displayname` と `#username` にコピーする。
2. ただし、補完先フィールドに **ユーザーが手動で入力した値** が残っている場合は上書きしない。
   - 判定方法: フィールドの現在値が「空」または「このスクリプトが直前に自動セットした値」と一致する場合のみ上書き対象とする。
   - 手動編集された時点でその値を保持し続け、以降メールアドレスを変更しても再度上書きされない。
3. `#username` を書き換えた場合は `dispatchEvent(new Event("input"))` を発火させ、SMTP設定側の同期ロジック（`initSMTPSync`）にも変更を伝播させる。

## 実装上の注意

- `#displayname` と `#username` それぞれについて、直近の自動セット値を `lastAutoDisplayName` / `lastAutoUsername` という変数で個別に保持している。片方だけ手動編集された場合でも、もう片方は自動補完を継続する。
- スクリプトは `#email` / `#displayname` / `#username` のいずれかが DOM に存在しない場合は何もしない（`guide.astro` など、フォームを持たないページでのエラーを防ぐため）。
- 「アカウントの説明」のような追加フィールドは対象外。現状は差出人名・受信ユーザー名の2つのみを補完する。

## 動作確認例

1. `#email` に `taro@example.com` と入力 → `#displayname` と `#username` に同じ値が自動反映される。
2. `#displayname` を `山田太郎` に手動で書き換える。
3. `#email` を `taro2@example.com` に変更 → `#displayname` は `山田太郎` のまま変わらず、`#username` は `taro2@example.com` に更新される（まだ手動編集されていないため）。
